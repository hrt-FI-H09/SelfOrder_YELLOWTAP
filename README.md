# SelfOrderSystem_YELLOWTAP

海外のストリートスタンドのようなカジュアルポップな世界観と、最先端の使いやすさを融合させた、モダンなクラフトビールパブ向けのタブレット注文UI/UXデザインプロジェクトです。

単に見た目が美しいデザインを作るだけでなく、フロントエンドへの実装（HTML/CSS、ReactのFlexbox設計）を1対1で直訳できるように、Figmaのコンポーネント構造・命名規則を論理的に組み立てています。

---

## プロジェクトの概要

- **ターゲット:** 20代〜30代のカジュアルにクラフトビールを楽しみたい層
- **プラットフォーム:** 店内設置型タブレット（16:9 横画面レイアウト）
- **主要な画面機能:** ビール・フードのメニュー選択、味の特徴（甘口・辛口）のスライダーインジケーター、おすすめのセットメニュー提案、詳細モーダル、注文管理

---

## 本プロジェクトにおけるUI/UXデザインの強み

### 1. 開発（フロントエンド）とシームレスに繋がる命名規則・コンポーネント設計
Figma内のすべてのレイヤーやフレームは、フロントエンドでのコーディング（BEM規則やコンポーネント指向開発）を意識して厳格に命名しています。
- **ハイフン（`-`）とスラッシュ（`/`）の明確な使い分け**
  - アセットパネルでのフォルダ階層分けが必要なマスターコンポーネントにのみ `/` を使用（例：`component/menu-card/default`）。
  - 通常のAuto Layoutフレームにはハイフンを使用（例：`taste-indicator`, `recommend-card`）し、CSSクラス名への直感を担保。
- **役割に応じた具体的な命名**
  - フレーム内で固有の意味を持つ要素は単なる `text` ではなく `title`, `price` と明記し、親フレーム名で役割が完結するものはシンプルに `text`, `dot` とすることで、データ構造の軽量化と可読性を両立。

### 2. コンポーネントの入れ子（ネスト）構造とレスポンシブ対応
Auto Layoutをフル活用し、すべての要素を論理的に入れ子構造化しています。
- セットメニューのミニカード（`recommend-card`）など、サイズ変更が予想されるエリアには全階層で連鎖的に `Fill container` を適用し、親要素の伸縮に滑らかに追従するレスポンシブな動きを完全再現。
- UIパーツの一般的な標準規格（スライダーにおける `thumb` や `track` など）に準拠した命名を行い、開発者とのコミュニケーションコストを最小限に抑える設計。

### 3. 世界観を引き立てるダイナミックなレイアウト設計
「YELLOW TAP」のブランドアイデンティティであるジューシーイエローと力強いフォント（Bebas Neue）をベースに、型に嵌まらないUIギミックを搭載しています。
- **絶対配置（Absolute position）の活用:** 詳細モーダルでは、主役であるビールのグラス写真をカードの左端から大胆にはみ出させ、中心線から右側に向かって情報が広がるダイナミックなレイアウトを実現。
- **コレクションカード風ミニメニュー:** 横幅が狭いグリッドでも情報が破綻しないよう、セットメニューは「上写真（1:1正方形ピタ留め）・下文字（中央揃え）」の縦型構造（`Vertical Auto Layout`）を採用し、限定ステッカーを並べて選ぶようなポップな楽しさを演出。
